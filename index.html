What a harness is

A harness is the boundary and interface around model capability. It decides what the model can see, which tools it may call, what memory is visible, what state can change, which actions require approval, and how outputs are checked before they become effects.

IO contracts: structured task inputs, output formats, schemas, and evidence requirements.

Tool gates: callable capabilities constrained by policy, sandbox, and action class.

Context policy: retrieval, redaction, memory scope, and prompt-injection resistance.

Verification: tests, lint, browser checks, diff review, confirmations, and logging.

5 / 10

Execution is a traceable flow

OpenClaw makes agent work inspectable by turning a request into a bounded execution trace: intake, context assembly, planning, delegation, action, verification, and durable capture.

  1. Normalize the task into intent, constraints, and side-effect class.
  2. Assemble only the context the harness policy permits.
  3. Choose local execution, delegated subagents, or external capabilities.
  4. Verify with the smallest meaningful gate before publishing state or effects.

Nodes expose device capabilities under policy

OpenClaw can treat phones, screens, browsers, cameras, files, and messaging surfaces as addressable capabilities. The important property is not breadth; it is that each capability is mediated by explicit permissions and a task-specific invocation path.

Screen

Observe UI state, record bounded evidence, inspect rendered artifacts.

Camera

Browser

Messages

Read, draft, or reply only through connector rules and approvals.

8 / 10

Persistence separates trace from memory

Not every token belongs in long-term memory. OpenClaw keeps raw task traces, session state, daily notes, durable decisions, and curated memory as distinct persistence layers with different promotion rules.

Trace: commands, tool outputs, diffs, screenshots, checks, and temporary reasoning artifacts.

Session state: active task plan, agent roles, open handles, pending approvals.

Daily log: factual events and work summaries worth retaining for near-term continuity.

Curated memory: durable preferences, decisions, lessons, and operating constraints.

9 / 10

Failure is modeled, not hand-waved

An agent runtime has to assume hallucination, stale state, tool failure, adversarial content, and accidental side effects. The harness contains those failures by narrowing scope, preserving evidence, and requiring validation before trust is spent.

Hallucination

Evidence-required claims, citations to local outputs, refusal on missing facts.

Tool error

Exit-code checks, bounded retries, surfaced uncertainty, no hidden mutation.

Stale state

Fresh reads before decisions, hash navigation, explicit timestamps and source age.

Prompt injection

Context provenance, instruction hierarchy, connector-specific trust boundaries.
